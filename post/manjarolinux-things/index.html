OSを変更した理由・背景

4月23日に、Ubuntu 20.04 LTSがリリースされた。
リリースに伴い、使用していたUbuntuのアップデートを行ったところ、snapを利用しインストールしていたソフトの入力周りに不具合が生じた。
aptを利用し、入れ直すなどを行って入力周りの不具合は解消できたが、バージョンアップ毎にこういった問題が起こるとなると少し不満。
また、Ubuntuの色合いは個人的に趣味に合わないため、気分転換も含めて別のLinux Distributionを見てみることにした。

なぜManjaroなのか

メジャーなDibian、Fedora、Slackwareに含まれない、独自の開発系統を持ったLinux Distributionである、Arch Linuxというのが良さげというのを聞いた。

Arch Linuxの利点や特徴については後述の参考資料に詳細なページへのリンクを記載するとして、自分がその中で響いたのは、ローリング・リリースを採用しており、最新バージョンのソフトウェアが使用できるという点だった。

上記よりArch Linuxが気になっていたが、導入の難易度が高く、自分の力量では心が挫けるのは目に見えていたため、
Arch系かつインストールが簡単であるManjaroを選択した。
見た目に関しても好みだった。

このManjaro、安定性についても気にかけたDistributionのようで、Arch Linuxとリポジトリを共有せず、独自のリポジトリを運用しているため、ド最新（今の現場でときたま聞く言葉）のソフトウェアを使用できるわけではないようだ。
といっても、概ね2週間程度のラグのため問題はとくに感じられない。

インストールに関して

Ubuntu18.04をインストールするときとほぼ変わらないぐらい簡単だった。
Arch Linuxのインストールガイドを見てビビっていたので、かなり驚いた。

困ったこと

日本語の設定がいろんなサイトで書かれているので、どれを参照すればよいのかイマイチだった。
Manjaroの公式Wikiにも日本語設定については書かれているんだけど、記載されている内容のパッケージが若干古かった。
以下は、自分用に日本語設定時に行ったことを記載しておく。

sudo pacman -S fcitx-im fcitx-mozc

~/.xprofileに以下を記載

export LANG="ja_JP.UTF-8"
export XMODIFIERS="@im=fcitx"
export XMODIFIER="@im=fcitx"
export GTK_IM_MODULE=fcitx
export QT_IM_MODULE=fcitx
export DefaultIMModule=fcitx

ログアウト、または再起動を行う。
